## Goal / Motivation

Add support for topology-aware selection of the nodes involved in the experiments

## Phases

### Phase 1 — Topology Parser

**Objective:**  Create a topology parser, which extract topology from the output of ```ibnetdiscover``` command

**Tasks:**

- [x] Create the parser as a python script, which outputs the network topology in a neutral representation format. The representation should be aware of concepts of nodes, NICs, switches and cells (nodes that are more than 2 switches away). You can use/refer to the code you can find at <https://github.com/cpt-harlock/cinetic>
- [x] Test the implementation against an example file

**Acceptance criteria:**

- [x] Correct parsing of an example file

**Implementation notes (done):**

- `src/crab/topology/model.py` — neutral, serializable representation: `Switch`, `Nic`, `Node`, `Cell`, `Link`, plus a `Topology` container with JSON `to_dict`/`from_dict`/`save`/`load` and a `Locality` enum (SAME_SWITCH / SAME_CELL / CROSS_CELL).
- `src/crab/topology/parser.py` — `parse_ibnetdiscover()` + argparse CLI. Regex grammar kept compatible with cinetic's `topology/topology.py`. Switches classified leaf/spine; NICs grouped into physical nodes by hostname; cells = connected components of leaves sharing a spine.
- `topology_parser.py` — root entrypoint (mirrors `cli.py`): `python topology_parser.py <ibnetdiscover.txt> -o topology.json`.
- `tests/fixtures/example_ibnetdiscover.txt` — synthetic 2-cell fabric; `tests/test_topology_parser.py` — runs standalone (`python tests/test_topology_parser.py`) or under pytest. All 6 tests pass.
- **Note for Phase 2:** CRAB models NICs as distinct from nodes (a node can own several HCAs), unlike cinetic which collapses both into one `CANode`.

**Open questions / decisions needed:**

- None for the moment

### Phase 2 — Topology Widget

**Objective:** in the TUI, when the user can select the nodes used for the experiment, add an option to select nodes in a graphical, topology-aware way. Again, you can use/refer to <https://github.com/cpt-harlock/cinetic> TUI.

**Tasks:**

- [x] Modify the TUI to add this new graphical way to select nodes
- [x] In the cluster presets, add a new option pointing to topology files, in the format defined in Phase 1. This topology files should be kept in a separated folder

**Acceptance criteria:**
I'll personally test the TUI  *(decided layout: graphical map)*

**Implementation notes (done):**

- `src/crab/tui/widgets/topology_map.py` — `TopologyMapScreen` (a `ModalScreen`) renders the fabric as a 2D map: cells → leaf-switch columns → clickable `NodeBlock`s (filled = selected). Clicking a cell/switch header bulk-toggles its nodes; buttons + keys for Select-All / Clear / Invert / Confirm / Cancel. Returns the sorted hostname list via `dismiss` (or `None` if cancelled).
- `BenchmarkOptions` (Benchmark Options tab): new **"Topology Map"** node source + **"🗺 Open Topology Map"** button. Opening loads the active preset's topology file; on confirm the chosen nodes fill the node table and set `numnodes`. The selection is also held on `self.selected_nodes`.
- Presets: new optional **`topology`** key (sibling of `env`/`sbatch`/`header`) pointing to a neutral-format file. `local` and `example_preset` point to `topologies/example.json`; default is `""` in `_common`.
- `topologies/` — folder holding neutral-format topology JSON (generated by Phase 1's parser).
- Styling appended to `src/crab/tui/assets/tui.tcss`. Headless pilot tests in `tests/test_topology_map_screen.py` (3/3 pass).

### Phase 3 — Explicit nodelist allocation

**Objective:** Pass the exact nodes chosen in the topology map (not just the count) to Slurm so the job is pinned to them.

**Tasks:**

- [x] Carry the selection from the TUI into the run/saved config as `global_options.nodelist` (list of hostnames). `BenchmarkOptions.get_state()` emits it; `set_state()` restores it on load.
- [x] In `engine.py`, emit `#SBATCH --nodelist=<hosts>` when a nodelist is present, and force `--nodes` to match its length. The nodelist is framework-managed: user `--nodelist`/`-w`/`--nodes` overrides are ignored (same protection model as `--nodes`/`--ntasks-per-node`).
- [x] Derive node count from the nodelist when `numnodes` is omitted (CLI users can supply just a `nodelist`).
- [x] `normalize_nodelist()` helper accepts a list or a comma/space-separated string, de-dupes preserving order.

**Acceptance criteria:**

- [x] `tests/test_engine_nodelist.py` (4 tests) covers header emission with/without a nodelist, count override, and the security check that users cannot override the managed nodelist.

**Notes:**

- The worker reads the actual `SLURM_NODELIST` of the allocation, so with `--nodelist` it already sees exactly the chosen hosts — no worker changes were needed.
- Hostnames are emitted verbatim (the topology selector produces explicit names, not Slurm range syntax like `node[01-04]`).
